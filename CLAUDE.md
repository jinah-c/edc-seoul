# 프로젝트 규칙 (CLAUDE.md)

## 기술 스택

- **Framework**: React + TypeScript
- **Router**: React Router DOM (`basename="/edc-seoul"`)
- **Bundler**: Vite (base: `/edc-seoul/`, port: 3000)
- **Icon**: FontAwesome (`@fortawesome/react-fontawesome`)
- **CSS**: 순수 CSS (CSS 변수 활용, 모듈 없음)

## 라우팅 규칙

- 모든 내부 링크는 `/edc-seoul/` prefix 없이 작성 (`/dispute/internet-apply`)
- React Router `basename="/edc-seoul"` 이 자동 처리
- 단, `href` 속성(a 태그)은 prefix 포함 필요 (`/edc-seoul/dispute/...`)
- `useNavigate` 사용 시 prefix 불필요

## CSS 규칙

### 변수 (variables.css)
```css
--primary-50 ~ --primary-700   /* 주 색상 */
--gray-100 ~ --gray-900         /* 그레이 */
--line-100                      /* 기본 선 색상 */
--font-small / --font-normal / --font-large
```

### 파일 관리
- **전역 스타일**: `src/styles/` 에 작성
- **페이지별 스타일**: 각 페이지 폴더 내 CSS 파일 (예: `Dma.css`)
- **같은 폴더 내 페이지들은 CSS 파일을 공유** (예: `003/` 의 모든 페이지는 `Dma.css` 사용)
- CSS 파일을 새로 만들지 말고 기존 파일에 통합

## 컴포넌트 규칙

### SubPageLayout
서브 페이지 기본 레이아웃. 반드시 사용.

```tsx
<SubPageLayout
  breadcrumbItems={[...]}
  lnbTitle="섹션명"
  lnbMenuItems={[...]}
  activeLnbId="현재메뉴id"
>
  <div className="r-content-wp">
    {/* 페이지 콘텐츠 */}
  </div>
</SubPageLayout>
```

### Breadcrumb
```tsx
const breadcrumbItems = [
  { label: "홈", url: "/" },
  { label: "상위메뉴", url: "#" },
  { label: "현재페이지" },  // url 없으면 현재 페이지
];
```

## 코딩 컨벤션

- 컴포넌트 파일명: PascalCase (`Dms-02.tsx`)
- CSS 클래스명: kebab-case (`.step-item`, `.form-radio-group`)
- 반복되는 JSX 요소는 파일 상단에 서브 컴포넌트로 분리
- 인터랙티브 요소에는 `aria-*` 속성 추가
- 탭 UI: `role="tablist"` / `role="tab"` / `role="tabpanel"` 사용
- 진행 단계 표시(stepper)는 탭이 아닌 `aria-current="step"` 사용

## 주의사항

- Dms.css 는 삭제됨 — `003/` 폴더 CSS는 모두 `Dma.css` 에 작성
- `.step-list` 의 step-item 은 클릭 불가 (진행 단계 표시용)
- 신청인 유형 탭 (개인/다수1가구/다수2가구이상) 은 클릭 가능한 탭
